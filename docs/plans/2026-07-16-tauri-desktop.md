# Tauri desktop build — design & plan

**Date:** 2026-07-16 · **Branch:** `desktop` (off `swiss-engine`)
**Decisions (user):** Claude executes; Windows is the primary target, Linux an extra;
plain wrapper + custom icon + window state memory + minimum window size.

## Design

Tauri v2 wraps the existing Vite app unchanged. The web build stays the source of
truth: `tauri dev` proxies the Vite dev server (`devUrl: http://localhost:5173`),
`tauri build` runs `npm run build` and embeds `dist/`. Nothing in `src/` changes —
the app already handles the desktop quirks (WASM is bundled and loads offline;
the Now button falls back when webview geolocation is flaky).

Since cross-compiling Tauri from Linux to Windows is not practical, Windows
installers (NSIS `.exe` + `.msi`) are built by **GitHub Actions** (`tauri-action`,
`windows-latest` runner). The same workflow builds Linux bundles (`.deb`,
`.rpm`, AppImage) on `ubuntu-22.04`. Locally on Arch, `tauri dev`/`tauri build`
work once `webkit2gtk-4.1` is installed (the only missing system package).

### Pieces

- `src-tauri/` — the Rust shell, hand-written (no interactive scaffold):
  - `Cargo.toml` — `tauri = "2"`, `tauri-plugin-window-state = "2"`, AGPL-3.0-only.
  - `src/main.rs` + `src/lib.rs` — default builder + window-state plugin
    (Rust-side only; no JS API, so capabilities stay at `core:default`).
  - `tauri.conf.json` — productName "Natal Chart", identifier
    `com.zagarov.natalchart`, window 1280×860, **min 1000×700** (332px panel +
    usable square wheel), `frontendDist: ../dist`, bundle targets `all`.
  - `capabilities/default.json` — `core:default` for the `main` window.
  - `icons/` — generated by `tauri icon` from a 1024px PNG rendered from a
    hand-made SVG in the app palette (ink sun-glyph wheel on parchment, gold ring).
- `vite.config.ts` — add `server.strictPort` so Tauri's fixed `devUrl` can't
  silently point at the wrong port; `clearScreen: false`.
- `.github/workflows/desktop.yml` — matrix (windows-latest, ubuntu-22.04);
  on tag `v*`: draft GitHub release with installers; on manual dispatch:
  build + upload artifacts only. Installers are **unsigned** — Windows
  SmartScreen will warn on first run (signing certs cost money; out of scope).
- `.gitignore` — `src-tauri/target/`, `src-tauri/gen/`.

## Tasks

1. Design doc (this file) committed on `desktop`.
2. `npm i -D @tauri-apps/cli`, add `"tauri"` script.
3. Write `src-tauri/` (Cargo.toml, build.rs, main.rs, lib.rs, tauri.conf.json,
   capabilities).
4. Icon: SVG → `rsvg-convert` → 1024px PNG → `npx tauri icon`.
5. `vite.config.ts` strictPort + `.gitignore`.
6. CI workflow.
7. Commit, push, trigger `workflow_dispatch`, verify the Windows build passes
   and produces installers. (Local `cargo check`/`tauri dev` verified after the
   user installs `webkit2gtk-4.1`.)
